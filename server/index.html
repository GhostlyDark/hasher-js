Hasher-js

Patching a ROM? Hasher-js computes ROM hashes to help you verify that you're patching the correct file. If you're creating a patch, use Hasher-js to get hashes of the original, unmodified file and share the hashes with your patch.

More Information

- hasher-js does not upload your ROM -- all the work is done in your browser on your machine
- hasher-js source is available at (...)
- a database match does not necessarily mean you have exactly the right file. hashes are the final word
- if you have the right ROM hash but the wrong file hash, you have the right ROM but in the wrong format
-- NES: Most likely garbage in the header, which is generally harmless, but certain header issues may manifest on patched ROMs on certain emulators
-- SNES: Could either be headered vs. headerless (can be remedied with a separate utility that can add or remove headers), or differences in external header (harmless)
-- Genesis: Most likely SMD vs. BIN format, which can be remedied with a separate utility.
-- N64: Most likely byte-swapped vs un-swapped, which can be remedied with a separate utility.
